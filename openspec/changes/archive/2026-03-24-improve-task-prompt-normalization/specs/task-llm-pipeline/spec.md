## MODIFIED Requirements

### Requirement: @task headlines routed to LLM pipeline
The system SHALL detect headlines tagged with `@task`. These headlines SHALL be routed to `sem-router--route-to-task-llm` instead of being silently discarded. The LLM SHALL return a single valid Org TODO entry with cleaned title, optional DEADLINE/SCHEDULED/PRIORITY, description, and a `:PROPERTIES:` drawer. The UUID for the `:ID:` field SHALL be pre-generated by Emacs and injected into the prompt. The Pass 1 prompt contract SHALL explicitly instruct the model to transform raw capture notes into complete structured TODO entries, and SHALL include runtime current datetime context for deterministic interpretation of relative time phrases.

#### Scenario: @task headline routed to LLM
- **WHEN** a headline has the tag `@task` (e.g. `* Task description :@task:`)
- **THEN** `sem-router.el` calls `sem-router--route-to-task-llm` with the headline content

#### Scenario: LLM returns structured TODO entry
- **WHEN** the LLM processes an `@task` headline
- **THEN** the response contains a valid Org TODO entry with title, properties drawer, and description

#### Scenario: Missing LLM priority tolerated for fallback
- **WHEN** the LLM response omits priority
- **THEN** the response remains acceptable to normalization flow
- **AND** downstream defaulting applies `[#C]` before planner stage

#### Scenario: Prompt includes runtime datetime anchor
- **WHEN** `sem-router--route-to-task-llm` constructs the Pass 1 prompt
- **THEN** prompt context includes runtime current datetime
- **AND** relative phrases in user content are interpreted against that anchor

### Requirement: Pass 1 asks for time range guess
The Pass 1 prompt SHALL request best-effort scheduling hints in format `SCHEDULED: <YYYY-MM-DD HH:MM-HH:MM>` when confidence is sufficient. The prompt SHALL also explicitly allow unscheduled output when timing intent is ambiguous.

#### Scenario: Pass 1 asks for time range
- **WHEN** Pass 1 prompt is constructed
- **THEN** it instructs the LLM to provide a provisional SCHEDULED with time range when timing confidence is sufficient

#### Scenario: Ambiguous timing allows unscheduled output
- **WHEN** Pass 1 cannot infer confident timing from normalized note content
- **THEN** output may omit `SCHEDULED`
- **AND** output remains valid for downstream planner handling
